# Behavioural-drift summary — 2026-07-03

First live run of `heirloom-tests-darwin` harness with a working Apout
Darwin arm64 build.

## Setup

- `heirloom-*-darwin` installed at `/opt/heirloom/` (per workspace ALM).
- Apout binary at `/opt/heirloom/vendor/apout/apout` (built from
  `moonman81/heirloom-apout-darwin` with the 2-patch series).
- V7 tape extracted at `/opt/heirloom/upstream-ancestors/v7/`
  (Henry Spencer donor tape from TUHS).

## Results

**9/9 comparisons PASS.** Heirloom-Darwin binaries produce
byte-identical output to their V7-1979 originals when run against
the corpora in `corpora/`.

| Tool | Corpus         | Result |
| :--- | :------------- | :----- |
| cat  | basic-sort.in  | PASS   |
| cat  | basic-uniq.in  | PASS   |
| cat  | basic-wc.in    | PASS   |
| echo | basic-sort.in  | PASS   |
| echo | basic-uniq.in  | PASS   |
| echo | basic-wc.in    | PASS   |
| tail | basic-sort.in  | PASS   |
| tail | basic-uniq.in  | PASS   |
| tail | basic-wc.in    | PASS   |

Each `reports/<tool>-<corpus>.diff` file contains the literal string
`PASS` (no diff to record).

## What this proves

For the three tools above, Ritter's Heirloom port preserves V7-1979
behaviour byte-exactly across the tested corpora. This is a real,
falsifiable demonstration of the preservation-fidelity claim that
Ritter's original project made — the first such demonstration in
the `moonman81/heirloom-*-darwin` universe.

## What this does NOT prove

- Only 3 tools tested — the harness needs many more (sort, wc, ls,
  date, etc. — see APOUT-STATUS.md for the V7-syscall-coverage
  limitations that block the rest).
- Only 3 corpora — behaviour on edge cases (empty input, huge input,
  binary data, non-UTF8) not yet catalogued.
- No property-based testing; the harness only checks the tested
  inputs.

## Next work

- Extend Apout's V7 syscall coverage (`v7trap.c` in
  `moonman81/heirloom-apout-darwin`) to make sort, wc, ls, date, sh
  work.
- Grow the corpora with edge-case inputs.
- Publish per-tool drift narratives when drift DOES appear.
