## Follow-up findings (added later 2026-07-04)

Extended the harness to 70 total comparisons.

### 8 col drift findings — CLOSED
Documented as SVR4 evolution.  `col -x` recovers V7 semantics.
See `DRIFT-col-tab-handling.md`.

### 3 factor drift findings — APOUT LIMITATION, not Heirloom drift
V7 `factor` via Apout produces empty output — the V7 tool needs a
syscall Apout does not fully emulate.  Heirloom `factor` produces
correct output.  This is an ARTEFACT of Apout's V7 syscall coverage
gap, not a Heirloom behavioural difference.  Filed under Apout
work-remaining (see moonman81/heirloom-apout-darwin APOUT-STATUS).

### 2 wc drift findings — APOUT LIMITATION, not Heirloom drift
Same as factor: V7 wc via Apout returns empty.

### 2 od drift findings — REAL BEHAVIOURAL DIFFERENCE (needs analysis)
V7 od default output format via Apout differs from Heirloom od default.
V7 output shows lower-byte values (0x09, 0x0a, 0x0c, 0x0d — the
control characters actually present in mixed-lines.in) while Heirloom
od shows word-format 6-digit octal values with byte-pair packing.

This may be:
  (a) A real behavioural drift in the od default format between V7
      and SVR4 (V7 might default to byte-oct, SVR4 to word-oct).
  (b) An Apout endianness / word-size interpretation issue.

Needs deeper analysis with -c (character) and -b (byte) flags to
disambiguate.  Recording as OPEN for now.
